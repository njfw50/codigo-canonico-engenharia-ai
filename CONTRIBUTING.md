Toda a gente é bem-vinda a contribuir para o desenvolvimento e evolução do Código Canónico de Engenharia de IA e Software. Se desejar contribuir, por favor, siga as diretrizes abaixo.

### Como Contribuir

1.  **Fork o repositório:** Comece por fazer um fork do repositório para a sua conta GitHub.
2.  **Crie um branch:** Crie um novo branch para as suas alterações. Use um nome descritivo para o branch (por exemplo, `feature/nova-lei-de-seguranca` ou `fix/correcao-lei-7`).
3.  **Faça as suas alterações:** Implemente as suas alterações, seja a propor uma nova lei, a corrigir um erro de digitação ou a melhorar a documentação.
4.  **Submeta um Pull Request:** Quando as suas alterações estiverem prontas, submeta um pull request para o branch `main` do repositório original.

### Padrões de Código e Documentação

*   **Estilo de Escrita:** Mantenha um estilo de escrita claro, conciso e profissional.
*   **Formato Markdown:** Todos os documentos devem ser escritos em Markdown.
*   **Mensagens de Commit:** Use mensagens de commit claras e descritivas.

### Propostas de Novas Leis ou Alterações

Se desejar propor uma nova lei ou uma alteração significativa a uma lei existente, por favor, abra uma issue no repositório para discutir a sua proposta antes de submeter um pull request. Isto permite que a comunidade discuta a proposta e forneça feedback.

Obrigado por contribuir para o Código Canónico de Engenharia de IA e Software!
